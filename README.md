## 学習目標

このプロジェクトを通じて、以下のSQLの基本概念を学習できます：

- **SELECT文の基礎**：データの取得、条件絞り込み、並び替え
- **集計関数**：COUNT、SUM、AVG、MAX、MINを使ったデータ集計
- **GROUP BY / HAVING**：データのグループ化と集計結果の絞り込み
- **JOIN**：複数テーブルの結合（INNER JOIN、LEFT JOIN）
- **DML**：INSERT、UPDATE、DELETEによるデータ操作
- **DDL**：CREATE TABLE、ALTER TABLE、DROP TABLEによるテーブル定義
- **サブクエリ / CASE式**：高度なクエリの記述
- **アンチパターン**：実務で陥りがちなミスの理解と回避

<h2 id="目標学習期間">目標学習期間</h2>

このカリキュラムの目標学習期間は**2週間**です。

**注意：** 学習の進捗には個人差があります。2週間で完了できなくても焦らず、理解を深めながら進めてください。

このカリキュラムは、プログラミングの基礎知識があることを前提としています。全8章・36問の演習を通じて、実践的なSQLスキルを身につけることを目指します。

<h2 id="環境構築">環境構築</h2>

実装を始める前に、必要なツールをインストールしてください。

### 必要なツール

1. **Git**
   - バージョン管理システム
   - インストール方法：
     - **Mac**：[Git公式サイト](https://git-scm.com/download/mac)からダウンロード、またはHomebrewで `brew install git`
     - **Windows**：[Git公式サイト](https://git-scm.com/download/win)からダウンロード
   - インストール確認：
     ```bash
     git --version
     ```
     バージョンが表示されればOKです。

2. **Node.js** (v18以上推奨)
   - JavaScriptの実行環境（テストの実行に使用します）
   - インストール方法：
     - [Node.js公式サイト](https://nodejs.org/ja/)からダウンロード（LTS版を推奨）
     - インストールすると、npm（Node Package Manager）も一緒にインストールされます
   - インストール確認：
     ```bash
     node --version
     npm --version
     ```
     バージョンが表示されればOKです。

3. **Docker Desktop**
   - PostgreSQLデータベースの実行に使用します
   - インストール方法：
     - [Docker公式サイト](https://www.docker.com/products/docker-desktop/)からダウンロード
   - インストール確認：
     ```bash
     docker --version
     docker compose version
     ```
     バージョンが表示されればOKです。

4. **Visual Studio Code（VSCode）**
   - コードエディタ
   - インストール方法：
     - [VSCode公式サイト](https://code.visualstudio.com/)からダウンロード
   - 推奨拡張機能：
     - **Japanese Language Pack for Visual Studio Code**：VSCodeを日本語化できる
     - **SQLTools**：SQL文の実行やハイライトが便利になる
     - **SQLTools PostgreSQL Driver**：SQLToolsでPostgreSQLに接続できる

### インストール後の確認

ターミナル（Mac）またはコマンドプロンプト（Windows）を開いて、以下を確認してください：

```bash
# Gitのバージョン確認
git --version

# Node.jsのバージョン確認
node --version

# npmのバージョン確認
npm --version

# Dockerのバージョン確認
docker --version

# 現在のディレクトリ確認
pwd  # Macの場合
cd   # Windowsの場合
```

### リポジトリのクローン

GitとVSCodeのインストールが完了したら、リポジトリをクローン（ダウンロード）します。

#### 1. クローン先のディレクトリに移動

作業したいフォルダに移動します。例：デスクトップ

**Macの場合：**
```bash
cd ~/Desktop
```

**Windowsの場合：**
```bash
cd C:\Users\ユーザー名\Desktop
```

#### 2. リポジトリをクローン

以下のコマンドでリポジトリをクローンします。

```bash
git clone https://github.com/Gennai2750/iRup-Xedge_SQL_curriculum.git
```

#### 3. プロジェクトフォルダに移動

```bash
cd iRup-Xedge_SQL_curriculum
```

#### 4. 依存関係のインストール

```bash
npm install
```

#### 5. VSCodeで開く

VSCodeを起動して「ファイル」→「フォルダーを開く」から `iRup-Xedge_SQL_curriculum` フォルダを選択してください。

#### 6. 練習画面の起動

Docker Desktopが起動していることを確認してから、以下のコマンドを実行します。
データベースの起動と練習用Webサーバーの立ち上げが自動で行われます。

```bash

```

## カリキュラム構成と学習ポイント

### 第1章：データの基本抽出（SELECTの基礎） — 7問

| 問題 | タイトル | キーワード |
|---|---|---|
| 1-1 | 全てのデータを取得する | `SELECT *` |
| 1-2 | 特定の列だけを取得する | `SELECT` |
| 1-3 | 条件に一致するデータを絞り込む | `WHERE`, `=`, `>=`, `<=` |
| 1-4 | 複数の条件を組み合わせる | `AND`, `OR` |
| 1-5 | 文字列の曖昧検索を行う | `LIKE` |
| 1-6 | データを特定の順番で並び替える | `ORDER BY` |
| 1-7 | 取得した文字列を結合・加工する | `CONCAT`, `AS` |

**学習ポイント：** SQLの最も基本となるSELECT文を学びます。WHERE句による条件指定、ORDER BYによる並び替えなど、データ取得の基礎を身につけます。

### 第2章：データの集計とグループ化 — 6問

| 問題 | タイトル | キーワード |
|---|---|---|
| 2-1 | データの件数を数える | `COUNT` |
| 2-2 | 重複を除外して種類を抽出する | `DISTINCT` |
| 2-3 | 数値の合計値と平均値を求める | `SUM`, `AVG` |
| 2-4 | 最大値と最小値を求める | `MAX`, `MIN` |
| 2-5 | カテゴリごとにデータをグループ化して集計する | `GROUP BY` |
| 2-6 | 集計した結果に対して条件を絞り込む | `HAVING` |

**学習ポイント：** 集計関数とGROUP BYを学びます。「件数を数える」「合計を出す」「グループごとに集計する」といった、実務でよく使う操作を習得します。

### 第3章：複数テーブルの結合（JOIN） — 5問

| 問題 | タイトル | キーワード |
|---|---|---|
| 3-1 | 2つのテーブルを内部結合する | `INNER JOIN` |
| 3-2 | 結合した結果に対して条件で絞り込む | `INNER JOIN` + `WHERE` |
| 3-3 | 片方のテーブルを基準に外部結合する | `LEFT JOIN` |
| 3-4 | 外部結合を使って「紐づくデータがないレコード」を見つける | `IS NULL` |
| 3-5 | 3つ以上のテーブルを結合する | 複数JOIN |

**学習ポイント：** 複数テーブルを組み合わせてデータを取得する方法を学びます。INNER JOINとLEFT JOINの違いを理解し、実務でのテーブル結合の感覚を掴みます。

### 第4章：データの追加・更新・削除（DML） — 4問

| 問題 | タイトル | キーワード |
|---|---|---|
| 4-1 | 新しいレコードを1件追加する | `INSERT` |
| 4-2 | 条件を指定して特定のデータを更新する | `UPDATE` + `WHERE` |
| 4-3 | 既存の数値を元に計算して更新する | `UPDATE` + 式 |
| 4-4 | 条件を指定して特定のデータを削除する | `DELETE` |

**学習ポイント：** データの登録・変更・削除を学びます。UPDATE/DELETEでは必ずWHERE句をつけることの重要性を理解します。

### 第5章：テーブルの作成と構造変更（DDL） — 3問

| 問題 | タイトル | キーワード |
|---|---|---|
| 5-1 | 新しいテーブルを適切なデータ型で作成する | `CREATE TABLE` |
| 5-2 | 既存のテーブルに新しい列を追加する | `ALTER TABLE ADD` |
| 5-3 | テーブルそのものを削除する | `DROP TABLE` |

**学習ポイント：** テーブルの作成・変更・削除を学びます。適切なデータ型や制約の設定方法を理解します。

### 第6章：高度なクエリ（応用編） — 3問

| 問題 | タイトル | キーワード |
|---|---|---|
| 6-1 | クエリの中で別のクエリを実行する | サブクエリ |
| 6-2 | 条件に応じて出力する値を分岐させる | `CASE` |
| 6-3 | 各カテゴリの中で最も値が大きいレコードを取得する | サブクエリ応用 |

**学習ポイント：** サブクエリやCASE式など、より複雑なクエリの書き方を学びます。これまでの知識を組み合わせた応用力を鍛えます。

### 第7章：エンジニアが陥りがちなミスとアンチパターン — 5問

| 問題 | タイトル | キーワード |
|---|---|---|
| 7-1 | 恐怖の全件更新・全件削除 | `WHERE` 忘れ |
| 7-2 | 意図しないデータの爆発 | クロスジョイン |
| 7-3 | `NULL` の罠 | `IS NULL`, 集計関数 |
| 7-4 | インデックスが効かない検索 | SARGable |
| 7-5 | `GROUP BY` での不正なカラム指定 | `GROUP BY` |

**学習ポイント：** 実務でのインシデントを防ぐためのアンチパターンを体験学習します。「やってはいけないこと」を安全に学べる章です。

### 第8章：実践総合演習 — 3問

| 問題 | タイトル | キーワード |
|---|---|---|
| 8-1 | 各顧客の合計購入金額を算出し、金額が高い順に表示する | JOIN + 集計 |
| 8-2 | 最も売上金額が高い商品を特定する | サブクエリ + 集計 |
| 8-3 | 部門ごとの平均給与と従業員一覧を同時に表示する | JOIN + 集計 |

**学習ポイント：** これまでに学んだ知識を総合的に活用する実践問題です。JOIN、集計、サブクエリを組み合わせた実務に近いクエリを書きます。

## 学習のコツ

1. **問題文をよく読む**
   - 期待される出力の行数やカラム名が問題文に書かれています
   - まずは問題の意図を理解してからSQLを書き始めましょう

2. **章の順番に進める**
   - 後の章は前の章の知識を前提としています
   - 分からない問題があったら、前の章を復習しましょう

3. **実際にSQLを書いて動かす**
   - 読むだけでなく、自分で考えてSQLを書いてみましょう
   - 間違えても大丈夫です。エラーメッセージから学べることが多いです

## トラブルシューティング

### データベースが起動しない

- Docker Desktopが起動しているか確認してください
- ポート5432が他のアプリケーションで使用されていないか確認してください
- 以下のコマンドでコンテナの状態を確認できます：
  ```bash
  cd db
  docker compose ps
  docker compose logs
  ```

### `npm install` が失敗する

- Node.jsが正しくインストールされているか確認してください
- `node --version` でv18以上が表示されるか確認してください

## レビューの方法

全ての問題を解き終わったら、レビューを依頼します。以下の手順に従って、プルリクエストを作成してください。

### レビューの流れ

1. **ブランチを作成**（`feature/受講者名（英語）`）
2. **変更をコミット・プッシュ**
3. **プルリクエストを作成**（タイトル：`SQLカリキュラム　受講者名`）
4. **Teamsでレビュー依頼**

### 重要な注意事項

**絶対にマージボタンを押さないでください！**

- プルリクエストを作成したら、**マージボタンは押さないでください**
- マージはレビュアー（講師）が行います
- 勝手にマージすると、他の人の作業に影響が出る可能性があります
- レビューが完了し、承認されたら講師がマージします

### Gitコマンド（初心者向け）

Gitの使い方が分からない方向けに、コマンドを順番に説明します。

#### 1. 現在のブランチを確認

まず、現在どのブランチにいるか確認します。

```bash
git branch
```

`* main` と表示されていれば、メインブランチにいます。

#### 2. 新しいブランチを作成

`feature/受講者名（英語）` という名前のブランチを作成します。
**ブランチ名は英語で記述してください。**

**例：受講者名が「山田太郎」の場合**

```bash
git checkout -b feature/YamadaTaro
```

**例：受講者名が「佐藤花子」の場合**

```bash
git checkout -b feature/SatoHanako
```

このコマンドで、新しいブランチを作成し、そのブランチに切り替わります。

#### 3. 変更を確認

どのファイルが変更されたか確認します。

```bash
git status
```

変更されたファイルが赤色で表示されます。

#### 4. 変更をステージング（追加）

変更したファイルをステージングエリアに追加します。
すべての変更を追加する場合：

```bash
git add .
```

特定のファイルだけを追加する場合：

```bash
git add exercises/01_select/1_1_select_all/answer.sql
git add exercises/01_select/1_2_select_columns/answer.sql
```

#### 5. コミット（変更を記録）

変更をコミット（記録）します。コミットメッセージは変更内容を簡潔に書きます。

```bash
git commit -m "SQLカリキュラム実装"
```

#### 6. ブランチをプッシュ（アップロード）

作成したブランチをリモートリポジトリにプッシュします。

```bash
git push origin feature/YamadaTaro
```

（受講者名の部分は自分の英語名に置き換えてください）

初めてプッシュする場合は、以下のメッセージが表示されることがあります：

```
fatal: The current branch feature/YamadaTaro has no upstream branch.
```

その場合は、以下のコマンドを実行してください：

```bash
git push -u origin feature/YamadaTaro
```

#### 7. プルリクエストを作成

GitHubのページで、プッシュしたブランチに対してプルリクエストを作成します。

1. GitHubのリポジトリページを開く
2. 「Compare & pull request」ボタンをクリック
3. タイトルに **`SQLカリキュラム　受講者名`** と入力
   - 例：`SQLカリキュラム　山田太郎`
4. 説明欄に実装内容を簡単に記載（任意）
5. 「Create pull request」ボタンをクリック

#### 8. Teamsでレビュー依頼

プルリクエストを作成したら、Teamsでレビュー依頼をしてください。

- プルリクエストのURLを共有
- 「SQLカリキュラムのレビューをお願いします」などのメッセージを送信

### 参考：Gitの基本概念

- **ブランチ（branch）**：作業の枝分かれ。メインのコードを壊さずに作業できる
- **コミット（commit）**：変更を記録すること。写真を撮るようなもの
- **プッシュ（push）**：ローカルの変更をリモート（GitHub）にアップロードすること
- **プルリクエスト（Pull Request）**：変更をレビューしてもらうための依頼
